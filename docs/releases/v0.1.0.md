# RepoBrain v0.1.0

First public release of RepoBrain.

## Added

- Local, Git-friendly repo memory stored in `.brain/` as reviewable Markdown plus frontmatter.
- Candidate-first memory capture and review flow with commands such as `brain capture`, `brain review`, `brain approve`, and `brain promote-candidates`.
- Task-aware context injection and routing with `brain inject`, `brain suggest-skills`, `brain route`, and `brain start`.
- Workflow presets for teams that want different automation levels: `ultra-safe-manual`, `recommended-semi-auto`, and `automation-first`.
- Agent adapter assets and integration guides for Claude, Codex, Cursor, and Copilot.
- Demo proof, evaluation assets, and case studies under `docs/` and `docs/demo-assets/`.

## Changed

- The default onboarding path is `brain setup`, which initializes `.brain/`, applies a workflow preset, and can install the matching low-risk Git hook.
- Publish automation now runs the full release verification chain before `npm publish` so tagged releases do not skip lint, typecheck, format checks, package smoke validation, or dry-run packaging.

## Fixed

- Packaged CLI validation is now part of the release gate, reducing the chance of a tarball that builds in source form but fails after packaging.
- The checked-in demo proof bundle has been refreshed to match the current CLI behavior and generated output.

## Verification

- `npm run build`
- `npm run lint`
- `npm run typecheck`
- `npm run format:check`
- `npm test`
- `npm run demo:proof`
- `npm run eval:proof`
- `npm run smoke:package`
- `npm pack --dry-run`

## Known Limits

- Real npm publish still depends on repository secrets and npm account permissions such as `NPM_TOKEN`.
- Final release confidence should still include checking the GitHub Actions matrix status after pushing the release commit and before creating the `v0.1.0` tag.
- Demo proof assets include generated timestamps, so re-running the proof scripts will produce reviewable doc diffs.
