# RepoBrain v0.2.3

Patch release for the streamlined maintainer publishing flow after `v0.2.2`.

## Changed

- Add `npm run release:verify` to run the full release gate in one command before publishing.
- Add `npm run release:publish`, backed by a dedicated publish script that chooses between GitHub trusted publishing and `NPM_TOKEN` fallback automatically.
- Update the publish workflow and maintainer docs so GitHub Actions can use trusted publishing by default while still supporting token-based fallback during rollout.

## Verification

- `npm run format:check`
- `npx vitest run test/publish-package-script.test.mjs test/package-manifest.test.mjs`
- `npm run release:verify`

## Notes

- `v0.2.3` does not change the public CLI contract for end users.
- This patch is focused on making RepoBrain releases easier to operate and easier to diagnose when npm publishing is not fully configured yet.
