# RepoBrain v0.1.0

RepoBrain 的首个公开版本。

## Added

- 以 `.brain/` 为载体、采用可审阅 Markdown 加 frontmatter 的本地仓库记忆层。
- 以 candidate 为先的记忆采集与审核流程，覆盖 `brain capture`、`brain review`、`brain approve`、`brain promote-candidates` 等命令。
- 面向任务的上下文注入与路由能力，覆盖 `brain inject`、`brain suggest-skills`、`brain route`、`brain start`。
- 三种工作流预设：`ultra-safe-manual`、`recommended-semi-auto`、`automation-first`。
- 面向 Claude、Codex、Cursor、Copilot 的适配资产与集成文档。
- 位于 `docs/` 与 `docs/demo-assets/` 下的 demo proof、评估结果与案例文档。

## Changed

- 默认上手入口为 `brain setup`，用于初始化 `.brain/`、应用 workflow preset，并可按需安装低风险 Git hook。
- 发布自动化现在会在 `npm publish` 前执行完整发布校验链路，避免打 tag 后跳过 lint、typecheck、格式检查、打包 smoke 验证与 dry-run 打包。

## Fixed

- 将打包后 CLI 验证纳入发布门禁，降低“源码目录可运行，但 tarball 安装后失效”的风险。
- 已刷新仓库内提交的 demo proof 资产，使其与当前 CLI 行为和生成输出保持一致。

## Verification

- `npm run build`
- `npm run lint`
- `npm run typecheck`
- `npm run format:check`
- `npm test`
- `npm run demo:proof`
- `npm run eval:proof`
- `npm run smoke:package`
- `npm pack --dry-run`

## Known Limits

- 真实 npm 发布仍依赖仓库密钥与 npm 账号权限，例如 `NPM_TOKEN`。
- 最终发布前，仍建议在推送 release commit 后、创建 `v0.1.0` tag 前，确认 GitHub Actions 矩阵全部通过。
- demo proof 资产包含生成时间戳，因此重跑 proof 脚本时会持续产生可审阅的文档 diff。
