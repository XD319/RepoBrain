# RepoBrain v0.2.2

Patch release for the Windows CI formatting regression discovered after `v0.2.1`.

## Fixed

- Add repository-level line-ending rules with `.gitattributes` so Windows CI checks out `src/`, `test/`, and other text files with LF endings.
- Eliminate the remote `Format check` failure that only reproduced on `windows-latest`.

## Verification

- `npm run format:check`
- `git ls-files --eol src test`
- `npm run build`
- `npm run typecheck`
- `npm run lint`
- `npm test`
- `npm run smoke:package`

## Notes

- `v0.2.1` remains the local patch that restored formatting in the conversation-start rollout.
- `v0.2.2` is the follow-up patch that makes the same formatting contract hold on GitHub Actions Windows runners.
