# RepoBrain v0.2.0

Smart conversation refresh release.

## Added

- New `brain conversation-start` command for later fresh conversations in the same session.
- Programmatic API export for conversation-start decision building.
- Decision tracing for `start` / `inject` / `skip` outcomes in JSON output.

## Changed

- Recommended workflow now uses `brain start` for the first conversation in a session, then `brain conversation-start` for later fresh conversations.
- Generated steering rules and adapter templates for Claude, Codex, Cursor, and Copilot now align with the smart refresh contract.
- RepoBrain activity state now records the latest context load metadata so refresh decisions can consider task, paths, modules, and session profile recency.

## Fixed

- Avoid redundant context reloads for same-session follow-up conversations when the active task context still matches.
- Keep explicit `brain inject` behavior compatible for users and adapters that still want direct lightweight context loading.

## Verification

- `npm run build`
- `npm run typecheck`
- `npm run lint`
- `npx vitest run test/conversation-start-command.test.mjs test/programmatic-api.test.mjs test/steering-rules.test.mjs`

## Known Limits

- Adapters still need to decide when a host tool has started a genuinely new conversation inside the same session.
- Full packaged smoke validation and proof asset refresh should still be rerun immediately before publishing the npm package.
