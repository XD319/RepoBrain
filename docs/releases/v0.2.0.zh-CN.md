# RepoBrain v0.2.0

这是一次围绕智能 conversation refresh 的版本发布。

## Added

- 新增 `brain conversation-start` 命令，用于同一 session 里后续新 conversation 的智能上下文刷新。
- 新增 conversation-start 的程序化 API 导出。
- JSON 输出新增 `start` / `inject` / `skip` 决策追踪信息。

## Changed

- 推荐工作流更新为：session 首个 conversation 用 `brain start`，后续新 conversation 用 `brain conversation-start`。
- Claude、Codex、Cursor、Copilot 的 steering rules / adapter 模板已同步到新的智能刷新契约。
- RepoBrain activity state 现在会记录最近一次 context load 的元数据，让刷新决策可以结合任务、路径、模块和 session profile 新鲜度。

## Fixed

- 当同一 session 的后续 conversation 与当前任务上下文仍然一致时，避免重复刷新上下文。
- 保持显式 `brain inject` 的兼容行为，方便仍然需要直接轻量上下文加载的用户和 adapter。

## Verification

- `npm run build`
- `npm run typecheck`
- `npm run lint`
- `npx vitest run test/conversation-start-command.test.mjs test/programmatic-api.test.mjs test/steering-rules.test.mjs`

## Known Limits

- 各 adapter 仍然需要自己判断宿主工具里“同一 session 的一个新 conversation”究竟在什么时候发生。
- 在真正发布 npm 包之前，仍然应该重新跑一遍完整的打包 smoke validation 和 proof asset 刷新。
