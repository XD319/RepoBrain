# RepoBrain v0.2.4

这是针对 `v0.2.3` tag 发布失败后，修复 GitHub Actions 发布路径的补丁版本。

## Fixed

- 在 GitHub Actions 中默认优先使用 npm Trusted Publishing，即使仓库里还保留着旧的 `NPM_TOKEN` secret 也不会误走 token 发布。
- 当发布脚本选择 trusted publishing 时，主动移除 token 认证相关环境变量，避免旧的 fallback 凭据悄悄劫持发布路径。
- 保留 token fallback 给本地维护者和显式的 CI 覆盖场景，而不是让它在自动模式下默认优先。

## Changed

- 收紧 release guide 和 checklist 的描述，让 CI 里通过 `NPM_TOKEN` 回退必须是显式选择，并通过 `REPOBRAIN_PUBLISH_STRATEGY=token` 触发。
- 扩充发布脚本测试，覆盖 GitHub Actions 下的策略优先级和认证环境变量处理。

## Verification

- `npm run format:check`
- `npx vitest run test/publish-package-script.test.mjs`
- `npm run release:verify`

## Notes

- `v0.2.4` 保留了 `v0.2.3` 引入的维护者发版命令。
- 这个补丁的重点是让远端 GitHub Actions 发布 workflow 真正走到预期的 trusted publishing 路径上。
