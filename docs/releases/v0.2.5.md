# RepoBrain v0.2.5

Patch release for the remote npm publish workflow after the failed `v0.2.4` tag run.

## Fixed

- Run the GitHub Actions publish workflow on Node `24` so the bundled npm CLI satisfies npm Trusted Publishing requirements.
- Keep the trusted publishing path aligned between the repository workflow and the maintainer-facing release guide/checklist.

## Changed

- Add an explicit release-checklist gate for the publish runner Node/npm version so future release prep catches this before tagging.

## Verification

- `npm run format:check`
- `npx vitest run test/publish-package-script.test.mjs test/package-manifest.test.mjs`
- `npm run release:verify`

## Notes

- `v0.2.5` does not change RepoBrain CLI behavior.
- This patch is narrowly focused on making the GitHub Actions publish workflow compatible with npm Trusted Publishing.
