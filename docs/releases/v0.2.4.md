# RepoBrain v0.2.4

Patch release for the GitHub Actions publish path after the failed `v0.2.3` tag run.

## Fixed

- Prefer npm Trusted Publishing by default inside GitHub Actions, even when an old `NPM_TOKEN` repository secret still exists.
- Strip token-based auth environment variables when the publish script selects trusted publishing, so stale fallback credentials cannot silently hijack the release path.
- Keep the token fallback available for local maintainers and explicit CI overrides instead of letting it win automatically.

## Changed

- Tighten release guide and checklist wording so CI fallback through `NPM_TOKEN` must be an explicit choice with `REPOBRAIN_PUBLISH_STRATEGY=token`.
- Expand publish script tests to cover GitHub Actions preference order and auth environment shaping.

## Verification

- `npm run format:check`
- `npx vitest run test/publish-package-script.test.mjs`
- `npm run release:verify`

## Notes

- `v0.2.4` keeps the maintainer release commands introduced in `v0.2.3`.
- This patch is specifically aimed at making the remote GitHub Actions publish workflow choose the intended trusted publishing path.
