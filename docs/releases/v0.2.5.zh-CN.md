# RepoBrain v0.2.5

这是针对 `v0.2.4` tag 远端发布失败后，修复 npm 自动发布 workflow 的补丁版本。

## Fixed

- 将 GitHub Actions 发布 workflow 切到 Node `24`，让随附的 npm CLI 满足 npm Trusted Publishing 的要求。
- 让仓库里的发布 workflow 与面向维护者的 release guide / checklist 保持一致，继续稳定走 trusted publishing 路径。

## Changed

- 在 release checklist 里增加发布 runner Node/npm 版本检查项，避免后续再次在打 tag 之后才暴露同类问题。

## Verification

- `npm run format:check`
- `npx vitest run test/publish-package-script.test.mjs test/package-manifest.test.mjs`
- `npm run release:verify`

## Notes

- `v0.2.5` 不改变 RepoBrain CLI 的对外行为。
- 这个补丁只聚焦于让 GitHub Actions 发布 workflow 真正兼容 npm Trusted Publishing。
